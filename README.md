## 功能特点

### 三大核心池子

1. **📋 待研究企业池**
   - 记录感兴趣但还未深入了解的企业
   - 适合收集招聘信息、企业动态

2. **📤 待投递岗位池**
   - 管理已研究完毕、准备投递的岗位
   - 记录岗位详情、薪资范围等信息

3. **✅ 已投递企业池**
   - 追踪已投递的岗位状态
   - 支持多种状态标记：已投递、已查看、面试邀请、已面试、已拒绝、已通过

### 核心功能

- ✨ **可视化看板**：三列式布局，清晰展示求职进度
- 📊 **实时统计**：顶部显示各池子的数量统计，包括面试中的岗位数
- ➕ **快速添加**：每个池子都可以快速添加新项目
- ✏️ **灵活编辑**：支持编辑任何项目的详细信息
- 🔄 **状态流转**：通过按钮或拖拽在不同池子间移动项目
- 💾 **本地存储**：数据自动保存在浏览器本地，无需服务器
- 🎨 **美观界面**：现代化设计，渐变背景，流畅动画

### ⚡ 快速录入功能

- **双模式识别**：
  - 本地识别：基于正则表达式，快速免费
  - AI 识别：调用 AI API，准确度更高
- **智能解析**：自动识别企业名称、岗位、地点、薪资、行业等
- **预览确认**：识别结果实时预览，确认后一键保存
- **支持格式**：
  - 企业名称 - 岗位名称
  - 地点：城市名
  - 薪资：XX-XXK
  - 行业：行业名称
  - 以及各种非标准格式（AI 模式）

### 📄 简历管理功能

- **多版本管理**：创建和管理多份简历（如：互联网版、金融版）
- **详细信息**：记录简历版本号、目标岗位、文件路径、备注
- **使用统计**：自动统计每份简历的使用次数和使用记录
- **快速选择**：投递时可选择使用的简历版本
- **关联追踪**：查看每份简历都投递了哪些公司

### 📈 投递进展功能

- **数据统计**：
  - 总投递数量
  - 已查看数量
  - 面试邀请数量
  - 已通过数量
  - 已拒绝数量
- **时间线展示**：按时间倒序展示所有投递记录
- **详细信息**：显示投递日期、使用的简历、当前状态
- **进度追踪**：清晰了解每个投递的最新进展

### 信息管理

每个项目可以记录：
- 企业名称（必填）
- 岗位名称
- 所属行业
- 工作地点
- 薪资范围
- 投递状态（仅已投递池）
- 使用简历（仅已投递池）
- 投递日期（仅已投递池）
- 备注信息

## 快速开始

### 第一步：打开应用

直接在浏览器中打开 `index.html` 文件即可使用。

### 第二步：配置 AI（可选）

如果想使用 AI 智能识别功能：

1. 点击右上角"⚙️ 设置"
2. 选择 AI 服务提供商（推荐 DeepSeek，国内可用且便宜）
3. 填入 API Key（参考《AI配置指南.md》获取）
4. 点击"保存配置"和"测试连接"

**不配置 AI 也可以正常使用**，本地识别功能完全免费。

## 使用方法

### 基础操作

1. **打开应用**
   - 直接在浏览器中打开 `index.html` 文件

2. **添加企业/岗位**
   - 点击每个池子顶部的"+ 添加"按钮
   - 填写相关信息后保存

3. **移动项目**
   - 方式一：点击卡片上的箭头按钮
   - 方式二：直接拖拽卡片到目标池子

4. **编辑/删除**
   - 点击卡片上的编辑（✏️）或删除（🗑️）按钮

### 快速录入

1. 点击顶部"⚡ 快速录入"按钮
2. 复制招聘信息并粘贴到文本框
3. 点击"🔍 识别信息"按钮
4. 确认识别结果后点击"保存"

**示例输入格式：**
```
阿里巴巴 - 高级前端工程师
地点：杭州
薪资：25-40K
行业：互联网
要求：3年以上经验，熟悉React/Vue
```

### 简历管理

1. 点击顶部"📄 简历管理"按钮
2. 点击"+ 新建简历"创建简历
3. 填写简历名称、版本、目标岗位等信息
4. 投递时可在"已投递企业"表单中选择使用的简历
5. 在简历详情页查看使用统计

### 投递进展

1. 点击顶部"📈 投递进展"按钮
2. 查看各状态的统计数据
3. 浏览时间线了解投递历史
4. 追踪每个投递的最新状态

## 技术栈

- 纯前端实现，无需后端服务器
- HTML5 + CSS3 + JavaScript (ES6+)
- IndexedDB 数据库（替代 LocalStorage，更稳定可靠）
- AI API 集成（支持 OpenAI、DeepSeek、智谱 AI、本地模型）
- 响应式设计，支持移动端

## 数据安全

- 所有数据存储在浏览器本地 IndexedDB
- 不会上传到任何服务器（AI 识别时仅发送招聘信息文本）
- 支持一键导出/导入 JSON 文件
- 方便在不同电脑间迁移数据
- 建议定期导出备份

## 浏览器兼容性

支持所有现代浏览器：
- Chrome / Edge (推荐)
- Firefox
- Safari

## 使用建议

1. **待研究企业池**：看到感兴趣的企业就先加进来，定期整理
2. **待投递岗位池**：研究清楚后移到这里，准备好简历再投递
3. **已投递企业池**：投递后及时更新状态，方便跟进面试进度
4. **快速录入**：浏览招聘网站时，直接复制信息快速录入，提高效率
5. **简历管理**：针对不同行业/岗位准备不同版本的简历，投递时选择最合适的
6. **定期查看进展**：每周查看投递进展，及时跟进面试机会

## 最佳实践

### 工作流程建议

1. **信息收集阶段**
   - 使用快速录入功能批量添加感兴趣的企业到"待研究"池
   - 每天花10-15分钟浏览招聘网站，快速录入

2. **研究筛选阶段**
   - 定期查看待研究企业，了解企业背景、文化、发展
   - 筛选出合适的岗位，移动到"待投递"池

3. **准备投递阶段**
   - 在简历管理中准备针对性的简历版本
   - 为不同类型的岗位准备不同的简历

4. **投递跟进阶段**
   - 投递时记录使用的简历和投递日期
   - 定期更新投递状态
   - 通过投递进展功能追踪整体情况

### 简历版本建议

- **通用版**：适合大多数岗位的基础版本
- **行业版**：针对特定行业（如互联网、金融、教育）
- **岗位版**：针对特定岗位类型（如技术、产品、运营）
- **公司版**：针对特别想去的公司定制
