## 完整工作流程示例

### 第一周：信息收集

**周一到周五（每天15分钟）**

1. 浏览Boss直聘、拉勾网、猎聘等平台
2. 看到感兴趣的岗位，复制信息
3. 使用"快速录入"功能，批量添加到"待研究企业"池
4. 本周目标：收集30-50个潜在机会

**操作步骤：**
```
1. 打开招聘网站
2. 复制岗位信息
3. 点击"⚡ 快速录入"
4. 粘贴信息 → 点击"识别信息" → 保存
5. 重复步骤2-4
```

---

### 第二周：研究筛选

**周末（2-3小时）**

1. 打开"待研究企业"池，逐个查看
2. 了解企业背景、文化、产品、发展前景
3. 评估岗位匹配度
4. 将合适的岗位移动到"待投递岗位"池
5. 删除不合适的企业
6. 本周目标：筛选出15-20个目标岗位

**筛选标准：**
- ✅ 岗位要求与自己匹配度 > 70%
- ✅ 薪资符合预期
- ✅ 企业发展前景好
- ✅ 工作地点可接受
- ✅ 企业文化认同

---

### 第三周：准备简历

**周末（半天）**

1. 打开"📄 简历管理"
2. 分析待投递岗位的类型
3. 准备3-4个版本的简历：
   - 互联网大厂版
   - 传统行业版
   - 创业公司版
   - 外企版

**简历准备清单：**
```
□ 更新项目经验
□ 调整技能关键词
□ 优化排版格式
□ 转换为PDF格式
□ 检查错别字
□ 请朋友review
```

---

### 第四周：开始投递

**每天投递3-5个岗位**

1. 从"待投递岗位"池选择岗位
2. 选择合适的简历版本
3. 投递后移动到"已投递企业"池
4. 记录投递日期和使用的简历
5. 设置状态为"已投递"

**投递记录示例：**
```
企业：字节跳动
岗位：前端开发工程师
投递日期：2025-11-16
使用简历：互联网通用版 v2.3
状态：已投递
```

---

### 持续跟进

**每天查看进展（5分钟）**

1. 打开"📈 投递进展"
2. 查看是否有新的状态更新
3. 及时更新投递状态：
   - 收到查看通知 → 更新为"已查看"
   - 收到面试邀请 → 更新为"面试邀请"
   - 面试完成 → 更新为"已面试"
   - 收到offer → 更新为"已通过"
   - 收到拒信 → 更新为"已拒绝"

**状态更新操作：**
```
1. 在"已投递企业"池找到对应卡片
2. 点击"✏️"编辑按钮
3. 更新"投递状态"
4. 添加备注（如面试时间、面试官反馈等）
5. 保存
```

## 数据分析示例

### 每周回顾

**查看投递进展统计：**
```
总投递：25个
已查看：15个（查看率60%）
面试邀请：8个（面试率32%）
已面试：5个
已通过：2个（通过率8%）
已拒绝：10个
```

**分析要点：**
1. 查看率低于50% → 简历可能需要优化
2. 面试率低于20% → 岗位匹配度不够
3. 通过率低于5% → 面试技巧需要提升
4. 查看简历使用统计，了解哪个版本效果最好

## 高级技巧

### 技巧 1：批量快速录入

浏览招聘网站时，先把所有感兴趣的岗位信息复制到一个文本文件，然后集中时间使用快速录入功能批量添加。

### 技巧 2：定期清理

每两周清理一次：
- 删除已拒绝且无价值的记录
- 将长期无回应的岗位移回待研究池
- 更新企业信息和岗位状态

### 技巧 3：备注详细信息

在备注中记录：
- 岗位亮点（为什么想投）
- 投递渠道（官网、内推、猎头）
- 面试反馈
- HR联系方式
- 面试时间安排

### 技巧 4：简历版本命名规范

建议格式：`岗位_行业_版本号`
- 前端_互联网_v2.3
- 产品_金融_v1.5
- 全栈_创业_v1.2

### 技巧 5：设置投递节奏

- 周一到周三：投递5-8个（HR精力充沛）
- 周四周五：投递2-3个
- 周末：不投递，准备下周的简历和岗位

## 常见问题

**Q: 快速录入识别不准确怎么办？**
A: 识别后可以手动修改，或者直接使用传统的添加方式。识别功能只是为了提高效率，不是必须的。

**Q: 简历文件路径有什么用？**
A: 记录简历文件位置，方便投递时快速找到对应版本的简历文件。

**Q: 数据会丢失吗？**
A: 数据保存在浏览器本地存储中，只要不清除浏览器数据就不会丢失。建议定期截图或导出备份。

**Q: 可以在手机上使用吗？**
A: 可以，但建议在电脑上使用，体验更好。手机上主要用于查看进展和更新状态。

**Q: 如何导出数据？**
A: 点击"设置" → "数据管理" → "导出数据"，会自动下载 JSON 文件。

**Q: AI 识别需要付费吗？**
A: 需要配置 AI API Key，不同服务商费用不同。DeepSeek 最便宜（约 0.002 元/次），也可以使用免费的本地模型。详见《AI配置指南.md》。

**Q: 本地识别和 AI 识别哪个好？**
A: 本地识别免费快速，适合格式规范的信息；AI 识别准确度高，适合复杂或非标准格式。建议优先使用本地识别，复杂情况再用 AI。
